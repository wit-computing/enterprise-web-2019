Routing in React.

In this lab we add routing to the apps being developed over the previous two lab exercises. Only the Phone Catalogue and Hacker News apps require routing logic.

Phone Catalogue App.

The objective with this app is to display detailed information for a phone when the user clicks it on the main view - see example below:

The URL for the new view is parameterized, for example:

'http://localhost:3000/phones/droid-2-global-by-motorola', where droid-2-global-by-motorola is a phone identifier.

The general form of the URL is:

http://localhost:3000/phones/:id

For convenience, all phone specifications are stored in JSON files at phoneCatalogue/public/phoneSpecs/phones. For example, the file pathname for the sample phone mentioned above is: phoneCatalogue/public/phoneSpecs/phones/droid-2-global-by-motorola.json. Find it in your filesystem.

Setup.

From threeApps/phoneCatalogue folder, install the React Router package:

   $ npm install react-router-dom@4.2.2 --save

and also the Superagent packages:

  $ npm install  superagent@1.6.1 --save

Start the development server:

  $ npm start

Routing Requirements.

This app will support two URL routes:

  1. The root path (/) - Display the main view (already developed).
  2. /phones/:id - Display a detailed specification for a specific phone.

The PhoneDetail component.

Phone details are stored in JSON files, for example, see phoneCatalogue/public/phoneSpecs/specs/dell-streak-7.json. Our React app can access this data using a HTTP GET request to the development server. In components/phoneDetail/index.js, update the code as follows:

. . . . other imports . . . . 
import localCache from '../../localCache';
import request from 'superagent' ; 

class PhoneDetail extends Component {
    componentDidMount() {
        const url = `/phoneSpecs/phones/${this.props.match.params.id}.json`
        request.get( url, 
            (err, res) => {
              let json = JSON.parse(res.text);
              localCache.setPhone(json);
              this.setState({});
            }) ;
    } 

    render(){
        let display = <p>No phone details</p> ; 
        let phone = localCache.getPhone();
        if (phone) {
            display =  
              <div>
                  <h2>{phone.name} </h2>
                  <p>{phone.description}</p>
              </div> 
        }
        return (
            <div className="container-fluid">
              <div className="row">
                  <div className="col-md-10">
                      {display}
                  </div>
              </div>
            </div>
        );
    }
    export default withRouter(PhoneDetail);

Ignore any errors this produces for the moment. This code uses a simple custom class to cache the current phone being viewed. Create the file src/localCache.js and paste in the following:

  class LocalCache {
      constructor() {
          this.phone = null 
      }

      setPhone(phone) {
          this.phone = phone 
      }

      getPhone() {
          return this.phone
      }

  }
  export default (new LocalCache() );

In the PhoneDetail component the componentDidMount method uses the Superagent library to make the HTTP request. The asynchronous nature of this behavior means we need to force a re-rendering after a response is received. The built-in setState() method achieves this, even though the component has no actual state data.

import React, { Component } from 'react';

export default class Form extends React.Component {
    state = { comment: '', name: ''};

    handleCommentChange = (e) => {
        this.setState({comment : e.target.value});
    };

    handleNameChange = (e) => {
        this.setState({name: e.target.value});
    };

    onSubmit = (e) => {
        e.preventDefault();
        let comment = this.state.comment.trim();
        let name = this.state.name.trim();
        if (!comment ) {
            return;
        }
        this.props.commentHandler(comment,name );
        this.setState({comment: '', name: ''});
    };

    render() {
        return (
            <form  style={{marginTop: '30px'}}>
                <h3>Add a new comment</h3>

                <div className="form-group">
                    <input type="text"  className="form-control"
                        placeholder="Comment" value={this.state.comment}
                        onChange={this.handleCommentChange} ></input>
                </div>     
                <div className="form-group">
                    <input type="text"  className="form-control"
                        placeholder="Your name" value={this.state.name}
                        onChange={this.handleNameChange} ></input>
                </div>
                <button type="submit" className="btn btn-primary"
                    onClick={this.onSubmit}>Submit</button>
            </form>
        );
    }
}

We can now assemble the new components in to a page/view. Create components/commentPage.js and copy in the following:

import React, { Component } from 'react';
import { withRouter } from 'react-router-dom';
import _ from 'lodash'
import api from '../dataStore/stubAPI';
import CommentList from './commentList'
import Form from './commentForm'

class CommentPage extends Component {
    addComment = (comment, name) => {
        let pid =  this.getId() 
        api.addComment(pid,comment ,name );
        this.setState({});
    };

    incrementUpvote = (commentId) => {
        let pid =  this.getId() 
        api.upvoteComment(pid,commentId) ;
        this.setState({});
    };

    getId = () => parseInt( this.props.match.params.post_id, 10);

    render() {
        let pid = this.getId()
        let post = api.getPost(pid);
        let line = post.link ?
            <a href={post.link}>{post.title} </a> :
            <span>{post.title} </span> ;
        let comments = _.sortBy(post.comments, 
                (comment) => - comment.upvotes
        ); 
        return (  
          <div className="container">
            <div className="row">
            <div className="col-md-9 col-md-offset-1">
                <h3>{line} </h3>
                <CommentList comments={comments} 
                    upvoteHandler={this.incrementUpvote } />
                <Form post={post}  addCommentHandler={this.addComment} /> 
            </div>
            </div>
          </div>
        )
    }
}

export default withRouter(CommentPage);

Rerendering policy.

One way to improve the responsiveness of a React app is by eliminating unnecessary re-renderings.

Contact List App.

In the ContactList app, in src/components/contact.js, add the following statement at the start of the render method:

  console.log(`Contact - ${this.props.contact.name}` )

Start the app server (npm start) and open the Chrome Developer tools in the browser tab. Now try the 'add a new contact' feature. Notice that all the existing contacts are (unnecessarily) re-rendered.

Delete a contact. Again all remaining contacts are re-rendered unnecessarily.

Add the following lifecycle method to the Contact component:

shouldComponentUpdate(nextProps, nextState) {
      let currentContact = {
          name: this.state.name,
          address: this.state.address,
          phone_number: this.state.phone_number
      }
      let same = _.isEqual(nextProps.contact, currentContact)
      return !same || (nextState.status === 'edit' )
        || (nextState.status !== this.state.status)
  }

We also need an import statement at the top:

 import _ from 'lodash'

Now a re-rendering of a contact will only happen when:

  1. It's status has changed (e.g. from normal to delete) or
  2. It is in edit mode (user is changing a property) , or
  3. The new and old props are not the same.

Try the add and delete contact features again. No existing contacts are re-rendered this time.

On completion, from the threeApps/contactlist folder, type the following commands:

     $ git add -A
     $ git commit -m "Remove unnecessary re-rendering "

Legacy Code.

We will use JSCodeShift to upgrade any legacy React code, concentrating on the Phone Catalogue app only (you may wish to repeat the process for the other apps.).

First, install the toolkit:

  $ npm install -g jscodeshift

Create a new folder (away from the three app folders), called, say, codemods. In the terminal window, go to the codemods folder and type:

 $ git clone https://github.com/reactjs/react-codemod.git

This downloads the codemod scripts for transforming legact React code.

To use jscodeshift command we must specify:

  • The pathname of the transformer script.
  • The pathname of the legacy source code to be examined for transformation.
  • Arguments for the transformer

In the terminal go to threeapps/phoneCatalogue and type:

   $ jscodeshift -t [path to codemods]/react-codemod/transforms/pure-component.js ./src/  --useArrows=true --destructuring=true

where:

  • '[path to codemods]/react-codemod/transforms/pure-component.js' is the transformer script pathname. You need to substitute in the pathname to your codemods directory.
  • './src' is the legacy code.

Two file will be modified by jscodeshift:

  1. src/index.js
  2. src/components/phoneList/index.js

Start the development server:

   $ npm start

JSCodeShift did make a tiny syntax error in the transformation. In src/components/phoneList/index.js move the export statement to the end:

    const PhoneList = .... as before ....
       . . . . . . 
    export default PhoneList

The app should now be rendering correctly in the browser.

On completion, from the threeApps/phonecatalogue folder, type the following commands:

     $ git commit -m "Upgraded legacy code using jscodeshift "
